# 💡 使用示例 - 与 Claude Code 配合

本文件展示如何使用元素定位器与 Claude Code 协作解决实际问题。

---

## 示例 1：修改按钮样式

### 场景
你发现登录页面的"提交"按钮颜色太浅，需要加深。

### 操作步骤

1. **使用插件提取信息**
   - 激活插件
   - 点击"提交"按钮
   - 自动复制到剪贴板

2. **粘贴给 Claude**

```markdown
我在登录页面发现一个样式问题：

【元素定位信息】
**点击的元素**
- 类型：按钮
- 样式类：`submit-btn` `primary`
- ID：`login-submit`
- 显示文本：登录
- 框架：Vue

**位置线索**
- 路径：`form.login-form > div.form-actions`

**建议搜索**
- 关键词：`submit-btn`, `登录`, `login-submit`

【问题描述】
按钮背景颜色太浅（#e0e0e0），可读性差

【期望效果】
加深按钮背景色为 #0078ff，提高对比度和可读性
```

3. **Claude 的回复**

Claude 会：
1. 搜索项目中的 `submit-btn` 或 `Login.vue` 文件
2. 定位到按钮的样式定义
3. 修改颜色值
4. 显示修改结果

## 示例 3：调整布局

### 场景
输入框之间的间距不一致。

### 操作步骤

1. **使用插件提取多个元素的信息**

先点击第一个输入框，复制信息。再激活插件，点击第二个输入框。

2. **粘贴给 Claude**

```markdown
表单布局问题：

【第一个元素】
**点击的元素**
- 类型：输入框
- 样式类：`form-input`
- name：`username`

【第二个元素】
**点击的元素**
- 类型：输入框
- 样式类：`form-input`
- name：`email`

【问题描述】
两个输入框之间的间距不一致，第一个间距是 16px，第二个是 24px

【期望效果】
统一所有输入框的间距为 16px
```

3. **Claude 的处理**

Claude 会：
1. 搜索表单组件
2. 定位输入框的样式定义
3. 统一 margin 或 gap 值
4. 确保所有输入框使用相同的间距

### 操作步骤

```markdown

- 显示文本：Submit（硬编码）
```

## 示例 8：无障碍优化

### 场景
按钮缺少无障碍标签。

### 操作步骤

1. **使用插件提取信息**

2. **粘贴给 Claude**

```markdown
无障碍优化需求：

【元素定位信息】
**点击的元素**
- 类型：按钮
- 样式类：`icon-btn` `close-btn`
- 显示文本：×（仅图标）
- 框架：React
- 组件名：`Modal`

【问题描述】
关闭按钮只有图标（×），缺少 aria-label，屏幕阅读器无法识别

【期望效果】
添加 aria-label="关闭" 或 aria-label="Close"，提升无障碍访问性
```

3. **Claude 的处理**

Claude 会：
1. 定位到 `Modal.jsx`
2. 添加 `aria-label` 属性
3. 确保键盘可访问（ESC 键关闭）

## 💡 最佳实践

### 提示词模板

```markdown
【元素定位信息】
（粘贴插件输出）

【问题描述】
（清晰描述当前的问题）

【期望效果】
（描述你想要的结果）

【其他信息】（可选）
- 浏览器：Chrome 120
- 设备：桌面端 / 移动端
- 其他相关说明
```

### 提高效率的技巧

1. **批量提取**
   如果有多个相关元素需要修改，可以逐个提取，一次性发给 Claude

2. **提供上下文**
   除了元素信息，也可以说明页面的路由路径，帮助 Claude 更快定位

3. **截图辅助**
   如果问题比较复杂，可以同时提供截图，让 Claude 更直观理解

4. **明确需求**
   尽量清晰地描述期望效果，避免模糊的表述
